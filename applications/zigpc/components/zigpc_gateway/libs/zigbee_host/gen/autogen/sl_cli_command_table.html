ota-server notify u16 u8 u8 u8 u16 u16 u32 otaImageNotifyCommand
Sends an OTA Image Notify message to the specified destination indicating a new version of an image is available for download.
Arguments
  • u16The node ID (can be a broadcast address) to which this OTA Notify message should be sent
  • u8Target endpoint for the OTA Notify message (only meaningful for non-broadcast target destination)
  • u8Specifies which parameters to include in the OTA Notify cluster command payload (0 = jitter value only; 1 = jitter and manufacturer id; 2 = jitter, mfr id, and device id; 3 = jitter, mfr id, device id, and firmware version)
  • u8Corresponds to QueryJitter parameter in the OTA Upgrade cluster specification. Indicates whether the client receiving Image Notify Command should send in Query Next Image Request command or not.
  • u16Manufacturer ID for the image being advertised (should match the mfr ID in the OTA files header)
  • u16Image type ID for the image being advertised (should match the image type ID from the OTA files header)
  • u32Firmware version of the image being advertised (should match the version from the OTA files header)

Instructs a device to upgrade now.
Arguments
  • u16Short destination to send message
  • u8Endpoint destination to send message
  • u16Manufacturer ID for the image (0xFFFF for wildcard)
  • u16Image type for the image (0xFFFF for wildcard)
  • u32File version for the image (0xFFFFFFFF for wildcard)
